# Состояния потока, User Level Threads vs Kernel Level Threads

## Состояния потока

1. New - поток только что создан, происходит инициализация структур данных
2. Runnable - поток готов к выполнению, но ещё не выполняется
3. ON CPU - поток выполняется на одном из процессоров
4. Blocked - поток ождиает какого-либо события
5. Finished - поток закончил выполнение и был остановлен

```
Потоки используют одно адрессное пространство родительского процесса, поэтому необходима синхронизация по общим данных. Блокировка потока не должна преводить к блокировке процесса.
```

## User Level Threads vs Kernel Level Threads

### Потоки на уровне пользователя

* Все операции (создание, завершение, переключение) над потоками выполняются специальными библиотеками в пользовательском пространстве без участия ядра. Уровню ядра соотвествует один поток.
* Приемущества: быстрое переключение контекста, меньше наклодные ресурсы, гибкая настройка планирования, меньше накладные расходы.
* Недостатки: ограниченность в использовании нескольких ядер, блокирующие системные вызовы, отсутствие приоритетов на уровне ядра.

## Потоки на уровне ядра

* Каждому потоку сопоставляется структура данных в ядре (kernel threads). Создание, завершение, переключение потоков происходит с использованеим системных вызовов
* Приемущества: настоящая параллельность, каждый поток виден всем ядрам. Блокировка одного потока не останавливает дургие. Гибкое планирование на уровне системы.
* Недостатки: Более высокие накладные ресурсы, сложность реализации.
